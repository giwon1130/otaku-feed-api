# 장애 대응 (Runbook)

> **누가 봄**: DevOps, BE
> **언제 봄**: 알림, 사용자 신고
> **함께 봐야 할 문서**: [`배포.md`](배포.md), [`보안.md`](보안.md)

> 사고 발생 후 **복기로 채워나감**. 새 증상은 위 신규 섹션으로 추가.

## 모니터링

- ✅ Railway 자체 metrics (RAM/CPU/Network/Disk)
- ✅ Railway 로그 (실시간 + history)
- ❌ APM/Sentry — 미도입 (필요 시 도입)
- ❌ Uptime 모니터링 (UptimeRobot 등) — 도입 검토

## 알려진 증상 → 원인 → 대응

### `/health`가 5–10초 걸림
- **원인**: Hobby plan 5분 idle → sleep → cold start
- **대응**: 자연 (한 번 깨면 빠름)
- **예방**: 클라 `apiHealth` keepalive (이미 있음)

### `/health`가 응답 안 함 (timeout)
- **원인**: 컨테이너 죽음 또는 배포 실패
- **대응**:
  1. Railway 대시보드 → Deployments → 최근 build 상태 확인
  2. Build Failed면 로그 확인 → 코드 fix → re-push
  3. Build Success인데 실행 안 됨 → Logs → Spring 부팅 에러 확인
- **흔한 원인**:
  - Flyway 마이그레이션 실패 (V<N> SQL syntax 오류)
  - 환경변수 누락 (`JWT_SECRET` 등)
  - DB 연결 실패 (`PG*` 환경변수 깨짐)

### 모든 사용자가 401 받음 (인증 실패)
- **원인**: `JWT_SECRET` 회전 후 → 기존 토큰 invalid
- **대응**: 의도된 동작 (회전한 거)
- **사용자 영향 완화**: 클라에서 401 받으면 자동 재로그인 유도 (TODO)

### 잘못된 JWT에 500 응답
- **원인**: `JwtProvider.extractUserId` 파싱 예외 unhandled
- **대응**: 백엔드 fix됨 (ResponseStatusException(401)) — ADR-007
- **재발 시**: `JwtProvider.extractUserId` 확인

### DB connection pool 고갈 ("Unable to acquire connection")
- **원인**: HikariCP max=5 초과 동시 요청 또는 connection leak
- **대응**:
  1. `application.yml` hikari pool size 증가 (max 5 → 10)
  2. Railway PG add-on의 max_connections 확인 (default 100)
- **진단**: 로그에서 connection close 못 한 코드 추적

### 응답이 큰 페이로드인데 gzip 안 됨
- **원인**: 응답 크기 < 1KB (`min-response-size: 1024`)
- **대응**: 의도된 동작 (작은 본문은 압축 오버헤드가 더 큼)
- **변경하려면**: `application.yml` `min-response-size` 조정

### Railway 비용 급증
- **원인 후보**:
  1. 사용자 폭증 (호출 수 증가)
  2. 클라 측 캐시/TTL 동작 이상 (호출 빈도 증가)
  3. egress 폭증 (응답 크기 증가)
- **진단**: Railway 대시보드 → Metrics → Network/Compute 그래프
- **대응**: [`비용최적화.md`](비용최적화.md) 참고. Spending Limit 설정.

### Flyway 체크섬 mismatch
- **증상**: 부팅 실패, "Validate failed: Migration checksum mismatch"
- **원인**: 이미 적용된 V<N>__*.sql 파일을 수정함
- **대응**:
  1. 즉시: 변경 내용을 V<N+1>__*.sql 새 파일로 이동
  2. 원본 V<N>은 git revert로 원래 상태 복구
  3. push → 정상 부팅

## 대응 우선순위

| 영향 범위 | 대응 시간 |
|----------|----------|
| 모든 사용자 인증 실패 (회전 사고) | 즉시 — 옛 secret 복구 |
| 백엔드 다운 (모든 동기화 안 됨) | 1시간 내 |
| 일부 endpoint만 실패 | 1일 내 |
| 비용 알림 | 그날 안에 점검 |

## 인증 / 인가

### JWT
- **알고리즘**: HMAC SHA-256 (`Jwts.builder().signWith(key)`)
- **secret**: `otaku-feed.jwt.secret` 환경변수 (32+ chars)
- **만료**: 30일 (`expiration-hours: 720`)
- **재발급**: Refresh token 미구현. 만료 시 사용자 재로그인.
- **회전**: secret 변경 시 모든 토큰 즉시 invalidate ([환경변수.md](환경변수.md) "키 회전")

### JWT 검증
- `JwtProvider.parse(token)` — 서명 + 만료 자동 체크
- `extractUserId(token)` — UUID 추출, 잘못된 토큰이면 401 throw (ADR-007)
- Spring Security filter는 path matcher 수준만 — userId 추출은 controller에서

### SecurityConfig (요약)
- `permitAll`: `/health`, `/auth/**`
- `authenticated`: 그 외 모두 (실제 검증은 controller에서 JWT 파싱)

→ 실제로 SecurityFilter는 거의 형식적. controller가 직접 `extractUserId`로 검증.

### 비밀번호
- **저장**: BCrypt 해시 (Spring Security `BCryptPasswordEncoder`)
- **검증 정책**: 현재 길이 검증만 (TODO: 복잡도/blacklist)
- **재설정**: 미구현 (TODO)

### OAuth
- **Google**: ID token을 Google 공개키로 검증 (구현 위치 확인 필요)
- **Kakao**: access token으로 Kakao API 호출해 user info 검증
- 자동 가입 (oauth_provider/oauth_subject 저장)

## CORS

- 설정: `OTAKU_CORS_ALLOWED_ORIGINS` (기본 `*`)
- 모바일 앱이라 origin 의미 X — 와일드카드 OK
- 웹 클라이언트 도입 시 명시적 origin으로 변경

## HTTPS

- Railway가 자동 처리 (LB → 컨테이너는 HTTP)
- 클라이언트는 항상 HTTPS URL만 사용

## SQL Injection

- JdbcTemplate `?` 바인딩만 사용 (문자열 concat ❌)
- 모든 쿼리 검토 시 `"... ${var} ..."` 패턴 금지

## Rate Limit

- 현재 없음. 부하 작아서 우선순위 낮음.
- 도입 검토 (악용 패턴 발견 시):
  - Bucket4j (in-memory)
  - nginx 또는 Cloudflare 앞단

## DoS 방어

- 현재 단일 인스턴스라 단순 large body 공격에도 약함
- Spring Boot 기본:
  - Multipart max-file-size 1MB
  - JSON body 자동 제한 (Jackson)
- 명시적 추가 검토:
  - `server.tomcat.max-http-form-post-size`
  - `spring.servlet.multipart.max-request-size`

## 로깅

- 현재 Spring 기본 logback
- ⚠️ 로그에 PII가 그대로 찍힘 (이메일 등)
- 도입 예정: 로그 마스킹 (Logback custom converter) 또는 구조화 로깅 (SLF4J + JSON)

## 비밀 관리

- ❌ Git에 secret 커밋 금지 — `.gitignore` 확인
- ✅ 모든 secret은 Railway 환경변수에만
- ✅ 코드에선 `@Value("\${X}")` 또는 `${ENV}`로 주입

## 사용자 권리 (GDPR/개인정보)

- ✅ **데이터 조회**: `GET /auth/me`, `GET /swipes`, `GET /prefs`로 사용자가 직접 확인 가능
- ❌ **데이터 삭제**: 계정 삭제 endpoint 없음 (TODO)
- ❌ **데이터 export**: 별도 endpoint 없음 (필요 시 위 GET들로 대체 가능)
- ❌ **개인정보 처리방침 페이지**: 없음 (App Store 심사 시 필요)

## 감사 체크리스트

- [ ] `JWT_SECRET` 32+ chars 랜덤
- [ ] `application.yml`의 dev secret이 운영에서 안 쓰임
- [ ] 모든 보호 endpoint가 `extractUserId` 호출
- [ ] SQL은 `?` 바인딩만
- [ ] CORS origin이 운영 환경에서 적절
- [ ] BCrypt round 적절 (10+, Spring 기본 OK)
- [ ] 마이그레이션이 schema `otaku`에 격리
- [ ] DB add-on 자동 백업 활성

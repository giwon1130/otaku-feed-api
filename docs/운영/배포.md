# 배포

> **누가 봄**: DevOps, BE
> **언제 봄**: 첫 배포, Railway 설정 변경, 마이그레이션 추가
> **단일 진실의 원천**: 이 문서. Railway 자체 설정은 대시보드 캡처 부족 → 변경 시 여기 반영.

## 현재 상태

- **Railway** 자동 배포 (Hobby plan)
- **트리거**: `git push origin main` → Railway가 webhook 받음 → ~2분 후 라이브
- **빌드**: Dockerfile (multi-stage Gradle)
- **서비스**: Spring Boot Java 21 + PostgreSQL add-on

## 자동 배포 흐름

```
1. git push origin main
2. Railway webhook 수신
3. Dockerfile 빌드 (gradle build → JAR → distroless 이미지)
4. 새 컨테이너 배포 (zero-downtime, health 통과 후 traffic switch)
5. Flyway 마이그레이션 자동 실행 (V*__*.sql)
6. ~2분 후 라이브
```

## 첫 배포 (새 환경)

### 1. Railway 프로젝트 생성
- [Railway 대시보드](https://railway.com) → New Project → Deploy from GitHub repo
- `otaku-feed-api` 선택

### 2. PostgreSQL add-on 추가
- Project → New → Database → PostgreSQL
- `PG*` 환경변수가 자동으로 main 서비스에 주입됨

### 3. 환경변수 설정
[`환경변수.md`](환경변수.md) 참고. 최소:
- `JWT_SECRET` (32+ chars 랜덤)
- `KAKAO_OAUTH_CLIENT_SECRET` (Kakao 사용 시)
- `GOOGLE_OAUTH_CLIENT_ID` (Google 사용 시)

### 4. 도메인 연결 (선택)
- Settings → Domains → Generate Domain (자동 *.railway.app)
- 또는 Custom domain (CNAME 설정)

## 일상 배포

```bash
cd /Volumes/Dev/Playground/otaku-feed-api
# 변경 → 빌드 확인
./gradlew build -x test
# 커밋 → 푸시
git add -A
git commit -m "feat/fix/perf: ..."
git push origin main
```

Railway 자동 배포 ~2분.

## 마이그레이션 추가

1. `src/main/resources/db/migration/V<N>__<설명>.sql` 생성 (V 번호는 기존 최대 +1)
2. `IF NOT EXISTS` 사용 권장
3. 로컬 빌드 확인 (`./gradlew build -x test`)
4. push → Flyway가 부트 시 자동 실행

⚠️ **이미 적용된 마이그레이션은 수정 X**. 수정하려면 새 V<N+1> 추가.

## 검증

배포 후 confirm:

```bash
# 살아있나
curl https://otaku-feed-api-production.up.railway.app/health
# → {"status":"UP","application":"otaku-feed-api"}

# gzip 적용
curl -I -H 'Accept-Encoding: gzip' https://otaku-feed-api-production.up.railway.app/health
# → content-encoding: gzip

# 보호된 endpoint (401)
curl -X POST https://otaku-feed-api-production.up.railway.app/swipes/bulk \
  -H 'Authorization: Bearer fake' -H 'Content-Type: application/json' -d '{"swipes":[]}'
# → HTTP 401 (Invalid or expired token)
```

## 롤백

Railway 대시보드 → Deployments → 이전 build 선택 → "Restore" 또는 git revert + push.

⚠️ DB 마이그레이션은 forward-only (Flyway 기본). 롤백 SQL 필요하면 새 V<N+1>로 작성.

# 데이터 모델

> **누가 봄**: BE, FE (응답 매핑 확인 시)
> **언제 봄**: 새 컬럼 추가, 마이그레이션 작성, 스키마 이해
> **단일 진실의 원천**: 이 문서. 실제 SQL은 `src/main/resources/db/migration/V*__*.sql`.

## Schema

모든 테이블은 `otaku` schema 아래.

## 테이블

### `otaku_users`
사용자 계정.

| 컬럼 | 타입 | 제약 | 설명 |
|-----|-----|-----|-----|
| `id` | UUID | PK, default `gen_random_uuid()` | 사용자 ID |
| `email` | TEXT | NOT NULL UNIQUE | 로그인 이메일 |
| `password` | TEXT | NOT NULL → nullable (V2) | BCrypt 해시. OAuth 사용자는 NULL |
| `nickname` | TEXT | NOT NULL | 표시명 |
| `created_at` | TIMESTAMPTZ | NOT NULL default `now()` | 가입 시각 |
| `oauth_provider` | TEXT | NULL (V2) | 'google' / 'kakao' |
| `oauth_subject` | TEXT | NULL (V2) | OAuth 제공자의 user id |

**인덱스**: PK + `email` UNIQUE 자동.

### `otaku_user_prefs`
사용자별 좋아하는 장르.

| 컬럼 | 타입 | 제약 | 설명 |
|-----|-----|-----|-----|
| `user_id` | UUID | PK, FK → `otaku_users(id)` ON DELETE CASCADE | 사용자 1 : 1 prefs |
| `favorite_genres` | TEXT[] | NOT NULL default `'{}'` | 영어 장르명 배열 |
| `updated_at` | TIMESTAMPTZ | NOT NULL default `now()` | 마지막 업데이트 |

**참고**: 1:1 관계라 user_id 자체가 PK.

### `otaku_user_swipes`
사용자의 좋아요/패스 기록.

| 컬럼 | 타입 | 제약 | 설명 |
|-----|-----|-----|-----|
| `id` | UUID | PK, default `gen_random_uuid()` | row id (응답엔 안 나감, ADR-005) |
| `user_id` | UUID | NOT NULL, FK → `otaku_users(id)` ON DELETE CASCADE | |
| `anime_id` | INTEGER | NOT NULL | AniList Media id |
| `result` | TEXT | NOT NULL CHECK ('like'/'dislike'/'skip') | |
| `swiped_at` | TIMESTAMPTZ | NOT NULL default `now()` | |
| | | UNIQUE `(user_id, anime_id)` | 사용자 × 작품 1번만 (upsert로 갱신) |

**인덱스** (V3에서 추가):
- `idx_swipes_user_id` (user_id) — V1
- `idx_swipes_result` (user_id, result) — V1
- `idx_swipes_user_swiped_at` (user_id, swiped_at DESC) — **V3 ★**
- `idx_swipes_user_result_swiped_at` (user_id, result, swiped_at DESC) — **V3 ★**

V3 인덱스는 `findByUserId ORDER BY swiped_at DESC`를 인덱스 한 번 스캔으로 해결.

## DTO ↔ Domain 매핑

### `UserSwipe` (Domain)
```kotlin
data class UserSwipe(
    val id: UUID,           // 응답엔 안 나감
    val userId: UUID,
    val animeId: Int,
    val result: SwipeResult,  // enum
    val swipedAt: Instant,
)

enum class SwipeResult { like, dislike, skip }
```

### `SwipeResponse` (Controller DTO)
```kotlin
data class SwipeResponse(
    val animeId: Int,
    val result: String,       // enum.name
    val swipedAt: String,     // Instant.toString()
)
```

### `BulkSwipeRequest`
```kotlin
data class BulkSwipeRequest(
    val swipes: List<SwipeRequest>,
)
data class SwipeRequest(
    @field:NotNull @field:Min(1) val animeId: Int,
    @field:NotBlank val result: String,
)
```

## 마이그레이션 이력

| 버전 | 파일 | 변경 |
|-----|------|-----|
| V1 | `V1__init_schema.sql` | users / user_prefs / user_swipes 초기 |
| V2 | `V2__add_oauth_columns.sql` | oauth_provider/oauth_subject + password nullable |
| V3 | `V3__optimize_swipe_indexes.sql` | 정렬 인덱스 (user_id, swiped_at DESC) 등 |

## 새 마이그레이션 작성

⚠️ V 번호는 기존 최대 +1. 한 번 적용된 마이그레이션은 **수정 X** (운영 DB 깨짐 + 체크섬 mismatch).

```sql
-- src/main/resources/db/migration/V4__<설명>.sql

CREATE TABLE IF NOT EXISTS otaku_xxx (
    id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id   UUID NOT NULL REFERENCES otaku_users(id) ON DELETE CASCADE,
    ...
);

CREATE INDEX IF NOT EXISTS idx_xxx_user ON otaku_xxx (user_id);
```

⚠️ `IF NOT EXISTS` 사용 — Flyway repeatable 안전.

## DB 호스팅

- **Railway PostgreSQL add-on** — 자동 backup
- 환경변수 `PG*`로 자동 주입 ([환경변수.md](../운영/환경변수.md))
- schema는 `otaku` (default-schema in `application.yml`)
